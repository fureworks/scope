### Task 2.2: Fix reviewer detection in git.ts
- In `getOpenPRs()`, the `reviewRequests` field from GitHub can be empty array even when reviews are requested via `requestedReviewers`
- Add `reviewDecision` to the gh query fields (APPROVED, REVIEW_REQUIRED, CHANGES_REQUESTED)

- Run test → should pass
- Commit: "fix: detect reviewers via reviewDecision field (#22)"

### Task 2.3: Fix confidence note in prioritize.ts
- In `scorePR()`, the condition `!pr.reviewRequested && pr.ageDays > 2` triggers "no reviewer assigned"
- This is wrong when the PR has CHANGES_REQUESTED (reviewer exists but flagged issues)
- Add `reviewDecision` to PRInfo interface
- Fix: only show "no reviewer assigned" when `!pr.reviewRequested && !pr.reviewDecision`
- Run tests → should pass
- Commit: "fix: confidence note respects reviewDecision (#22)"

- Test: Score range should use 0-100
- Run test → should fail
- Commit: "test: graduated scoring expectations"

### Task 3.2: Implement graduated staleness scoring
- In `scorePR()`, replace step-function scoring with graduated:
  ```ts
  // Old: 9 for >14 days (51-day and 15-day score the same)
  // New: graduated scoring that increases with age
  agePoints = Math.min(30, Math.round(pr.ageDays * 0.6));
  ```
- Scale all other scores proportionally (review: 15, CI: 10, conflicts: 8)
- Update priority thresholds: now ≥ 25, today ≥ 12
- Run test → should pass
- Commit: "fix: graduated staleness scoring (#23)"

### Task 3.3: Add label-based scoring boost
- Add `labels` field to `PRInfo` interface
- Fetch `labels` in `getOpenPRs()` gh query
- In `scorePR()`, add label boost:
  ```ts
  const LABEL_BOOSTS: Record<string, number> = {
    'P0': 15, 'critical': 15, 'blocker': 15,
    'security': 12, 'compliance': 12,
    'P1': 10, 'bug': 8,
    'P2': 4, 'enhancement': 2,
  };
  ```
- Case-insensitive substring match on label names
- Run tests → should pass
- Commit: "feat: label-based scoring boost (#23)"
